# Executive Reporting Guide

Generate executive-friendly HTML dashboards from security assessment findings for monthly leadership review.

---

## Quick Start

### Step 1: Run Assessments

```bash
# Create output directory
mkdir -p ~/security-assessments/$(date +%Y-%m)

# Run AD configuration assessment
cd ~/.claude/skills/ADSecurityAssessment
bun run Tools/AssessADMisconfigs.ts /path/to/ad-config.json > ~/security-assessments/$(date +%Y-%m)/report-misconfigs.txt

# Run privilege analysis
bun run Tools/AnalyzePrivileges.ts /path/to/identity-data.json > ~/security-assessments/$(date +%Y-%m)/report-privileges.txt

# Run Azure AD assessment
bun run Tools/AuditAzureAD.ts /path/to/azure-ad-data.json > ~/security-assessments/$(date +%Y-%m)/report-azure.txt
```

### Step 2: Generate Executive Report

```bash
# Generate HTML dashboard
bun run Tools/GenerateExecutiveReport.ts \
  ~/security-assessments/$(date +%Y-%m) \
  "Your Organization Name" \
  ~/security-assessments/$(date +%Y-%m)/executive-report.html
```

### Step 3: View & Distribute

```bash
# Open in browser
open ~/security-assessments/$(date +%Y-%m)/executive-report.html

# Or copy to web server for leadership access
```

## Troubleshooting

### "No findings loaded"

**Cause:** Report files not in expected format or location

**Solution:**
```bash
# Check report files exist
ls -la /path/to/reports/

# Verify report format
head -50 /path/to/reports/report-misconfigs.txt

# Ensure files end in .txt
```

### "Security score is 100 but there are findings"

**Cause:** Parser not recognizing findings from report

**Solution:**
- Ensure report was generated by PAI assessment tools
- Check for unicode/encoding issues
- Verify emoji characters display correctly

### HTML doesn't render properly

**Cause:** Browser compatibility or corrupted file

**Solution:**
- Try different browser (Chrome, Firefox, Safari)
- Check file size (should be 50-200KB typically)
- Regenerate report

## Best Practices

### 1. Consistent Scheduling
- Run assessments same time each month
- Use consistent data export methods
- Document any scope changes

### 2. Historical Tracking
- Keep all reports (organized by month)
- Never overwrite previous reports
- Archive after 2 years (compliance retention)

### 3. Executive Presentation
- Send report 24h before meeting
- Prepare to discuss top 3 findings
- Have remediation plans ready
- Show month-over-month trends

### 4. Follow-Up Actions
- Assign owners to CRITICAL/HIGH findings
- Set realistic timelines
- Track in project management system
- Report progress next month

### 5. Data Security
- Reports contain sensitive information
- Encrypt during transmission
- Limit access to authorized personnel
- Follow data classification policies
